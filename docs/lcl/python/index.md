# Python sur mainframe

Python n'a pas vocation à remplacer COBOL, mais il s'impose de plus en plus comme
langage d'automatisation, d'intégration et d'outillage autour des applications
z/OS existantes. Cette section explique pourquoi, et propose des exemples de code
concrets pour démarrer.
